Отчеты в Quest vWorkspace 7.2

Блог компании КРОК
Насколько важна отчетность в ИТ? Большинство админов ей вообще не пользуются — работает, и отлично. Однако в больших компаниях построение красивых графиков «Кто? Куда? Откуда? Зачем?..» занимает важную часть в работе ИТ-отделов. Ведь практически на каждое совещание, а особенно на подведения итогов месяца-квартала-полугодия-года, настроятельно рекомендуется приходить с грасивыми картинками, на что же расходуется ИТ-бюджет :). Как раз о построении отчетов применительно к ПО Quest vWorkspace 7.2 мы сегодня и поговорим.
Напомню, что vWorkspace — это ПО для построения VDI-решений, в том числе Enterprise-уровня. До настоящего момента несколько огорчало отсутствие возможности вcтроенными средствами узнать, какие пользователи подключались, сколько было отказов, какие действия производил тот или иной администратор системы. Однако начиная с версии 7.2 у нас, наконец, появилась такая возможность!
Итак, у нас развернута VDI на базе vWorkspace. В качестве базы данных используется SQL. Теперь попробуем настроить отчетность (в терминах Quest — «Reporting»). Для этого, используя консоль управления vWorkspace Management Console, открываем меню «Database Configuration».





Затем создаем новую базу данных на SQL-сервере, где располагается БД самого ПО vWorkspace.





Далее вводим учетные данные администратора SQL (я использовал встроенного SA, но это совсем не обязательно) и нового пользователя Report Viewer (имя по умолчанию) для подключения к БД.



Отлично — БД отчетности создана!



После создания этой БД наша ферма vWorkspace начнет собирать данные для последующего создания отчетов.
Собственно для создания и просмотров отчетов нам необходима роль vWorkspace Reporting. Она автоматически устанавливается при выборе опции Simple в момент инсталяции vWorkspace или может быть установлена на любом ПК с помощью опции Advanced -> Reporting and Logging Role.





После установки в меню Пуск появляется ярлык Start -> Programs -> Quest Software -> vWorkspace -> Report Viewer.



При первом запуске будет предложено настроить подключение к БД отчетности.



Наконец, мы может открыть в качестве примера любой из отчетов (их 18 штук), которые поставляются вместе с ПО. Расположены эти отчеты в той же папке, где и собственно vWorkspace Sample Reporte Viewer (по умолчанию — C:\Program Files\Quest Software\vWorkspace\Reporting). Для примера откроем отчет об активности пользователей.



Здесь же можно посмотреть описание отчета и запросы к базе SQL, а так же экспортировать данные в файл формата csv.
Давайте посмотрим на графики, которые составлены на примере отчетов, экспортированных в Excel (все рисунки взяты из блога компании Quest (http://communities.quest.com/community/vworkspace)





Как видите, даже с отчетами «по умолчанию» можно показать начальству, на что потрачены деньги и что ИТ-отдел не зря кушает свой хлеб!
Всегда рад Вашим комментариям.
