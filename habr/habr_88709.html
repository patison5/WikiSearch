Тест скорости сшивки панорам MS ICE vs Autopano Giga 2.0.3 и Hugin 0.7.0

IT-компании
Небольшой тест по сравнению скорости сшивки панорам в только что вышедшей новой версии (писал об этом здесь) Microsoft Image Composite Editor 1.3.3 (скачать). Сравниваю с двумя популярными сшивалками — Hugin и Autopano Giga. Тесты — панорама 50 MPix (10 кадров) и панорама 81 MPix (18 кадров).

 

Железо: CPU Intel Core 2 Duo E8500 3.16 GHz, RAM 4 Gb, OS Windows XP Professional x86 SP3, ATI Radeon HD 4850 512 Mb.

Пример 1

10 кадров 2592 x 3888, панорама 12200 x 4300 (50 MPix).
Программа preview render всего отличия от настроек по умолчанию
Autopano Giga 2.0.3 0:08 3:30 3:38 cache 512 mb
memory use: high
Autopano Giga 2.0.3 0:08 3:58 4.06 memory use: high
Autopano Giga 2.0.3 0:08 4:05 4:13  
Hugin 0.7.0 1:33 1:43 3:16  
ICE 1.3.3 0:13 0:29 0:42  

Из этого теста выяснилось, что с параметрами по умолчанию Autopano Giga работает не самым эффективным образом. В следующем тесте использую корректировки в параметрах, которые привели в первом тесте к лучшему результату.

Пример 2

18 кадров 2592 x 3888, панорама 13500 x 6300 (81 MPix).
Программа preview render всего отличия от настроек по умолчанию
Autopano Giga 2.0.3 0:14 11:21 11:35 cache 512 mb
memory use: high
Hugin 0.7.0 2:43 2:41 5:24  
ICE 1.3.3 0:21 0:57 1:18  

Вывод

ICE дает существенный выигрыш по скорости сшивки по сравнению с Autopano Giga и Hugin. Ускорение относительно Hugin составляет 4.6x (10 кадров) и 4.1x (18 кадров), относительно Autopano Giga растет с увеличением сложности панорамы — 5.1x (10 кадров) и 8.9x (18 кадров).

P.S.

Хотелось проверить новый режим ICE для сшивки гигапискельных структурированных панорам, но к сожалению этот режим стабильно падает как минимум в Windows XP. Проблема воспроизведена разработчиками (см. на форуме ICE) и надеюсь в скором времени будет выпущен исправленный вариант.
