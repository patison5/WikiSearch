Google AJAX Search API — огромный баг?

Чулан
Посвящается разработчикам Google SOAP API...

Для тех кто не знает что это — это API, с помощью которого вы можете делать запросы к поисковой машине Google.
Штука, казалось бы очень полезная, т.к. регистрация ключей к SOAP Search API была прекращена более 3-х лет назад,
а парсинг HTML-выдачи имеет кучу недостатков кроме одного достоинства — идентичности данных получаемых пользователем, с идентичностью данных получаемых программой.
Выходной формат данных JSON, что гораздо легче в использовании, чем SOAP, его также можно использовать client-side на JS или Flash.

Я познакомился с этим API около года назад, и очень сильно разочаровался в нем.
Почему спросите вы? Ответ прост — результаты выдаваемые через API очень сильно отличаются от выдачи поисковика через Web.
Хотя, по логике вещей они должны быть если не идентичны, то очень близкие по значению.

Но, увидев что в данный момент они не в production, я искренне понадеялся что к моменту выхода из Labs проблемы будут решены.
Итак, AJAX Search API вышел в production, SOAP Search API будет закрыт 31-го августа, т.к. по мнению Google не нужен.

Я решил проверить как обстоят дела, и сравнил текущую выдачу поисковика, SOAP API и AJAX API.
В качестве объекта сравнения выступает количество найденных страниц по поисковому запросу, ключевые слова специально не отбирались.

Кому лень смотреть — результаты очень сильно отличаются, что сводит ценность Google AJAX Search API к нулю.
